<decisions>
## Implementation Decisions

### Execution order
- Run sources fastest-first: 1) EPC Register + Companies House (scripts ready) 2) HMO registers (local CSV files in `HMO registers/` folder) 3) CCOD (auto-download from gov.uk) 4) OpenRent + Rightmove (Apify actors)
- After all sources ingested: run deduplication, then build-lead-list.js to produce real CSV exports
- Full end-to-end pipeline: gather all data -> dedup -> export CSVs with real data

### EPC and Companies House
- Execute `scripts/epc-api.js` and `scripts/companies-house.js` to populate DB with real data
- These scripts are already built and tested — just need to be run against live APIs
- API keys already registered and in .env

### CCOD (Land Registry)
- Claude auto-downloads the CCOD CSV from gov.uk (use Firecrawl or direct download)
- Then runs `scripts/import-ccod.js` with the downloaded file path
- Streaming importer handles the ~500MB file

### HMO registers
- Sam has local CSV files in `HMO registers/` folder (Stockport and Manchester councils)
- Run `scripts/parse-hmo.js` against these files
- No scraping needed — files already available

### OpenRent scraping
- Search Apify Store for OpenRent actors — need listings AND landlord profiles
- Capture: landlord name, property address, rent, listing date, photos, description quality
- Also follow landlord profile links to get their full portfolio of listed properties
- Geographic filter: SK1-SK8, M14, M19, M20, M21, M22 postcodes
- Process results through `scripts/process-openrent.js`

### Rightmove/Zoopla scraping
- Search Apify Store for Rightmove rental listing actors
- Focus: rental listings for void period analysis and listing quality scoring
- Capture properties listed 30+ days (void signal), poor photos, minimal descriptions
- Landlord identity comes from address-matching against other sources, not from Rightmove directly
- Both Rightmove AND Zoopla — not just one platform
- Process results through `scripts/process-rightmove.js`

### Tool priority chain
- Apify actors first choice for web scraping
- Firecrawl as fallback if Apify actors don't work well or get delisted
- Playwright as last resort if both above fail
- Each tool tried in order before escalating

### Google Maps Street View
- NOT captured in this phase — deferred to after Phase 3 scoring
- Only top-scored leads get Street View qualifying (saves API calls)
- Output: Google Street View URL column added to lead export CSV
- Sam reviews URLs manually in browser to assess exterior property condition
- Visual signals: peeling paint, overgrown garden, broken fences, dirty windows, damaged guttering, general neglect
- No image downloads — just URLs for manual review

### Pipeline orchestration
- No master orchestration script — Claude runs each step inline during execution
- More flexible, easier to debug, handles Apify actor setup interactively
- Each source run individually in sequence by Claude

### Claude's Discretion
- Specific Apify actor selection (search Store, pick best rated/most popular)
- Apify actor input configuration (search parameters, pagination)
- CCOD download URL and method
- Error handling and retry strategy per source
- Whether to batch postcodes or run all at once per source

</decisions>

## Existing Code Insights

### Reusable Assets
- `scripts/epc-api.js` — Ready to execute, produces landlord+property records from EPC Register API
- `scripts/companies-house.js` — Ready to execute, produces company landlord records from CH API
- `scripts/import-ccod.js` — Streaming CSV importer, just needs file path argument
- `scripts/parse-hmo.js` — CSV/XLSX parser with per-council column mapping
- `scripts/process-openrent.js` — Processes saved Apify JSON, inserts to DB
- `scripts/process-rightmove.js` — Processes saved Apify JSON with listing quality scoring
- `scripts/lib/dedup.js` — Two-pass exact+fuzzy deduplication engine
- `scripts/build-lead-list.js` — Produces 5 date-stamped CSV exports

### Established Patterns
- Plain JavaScript (Node.js), no TypeScript, no build step
- `require('dotenv').config()` for API keys
- Scripts in `scripts/`, data in `data/` (gitignored)
- SQLite at `data/sjb-leads.db` with landlords and properties tables
- Apify MCP tools available for actor discovery and execution

### Integration Points
- All scripts write to the same SQLite database via `scripts/lib/db.js`
- Apify actors produce JSON output that feeds into process-openrent.js and process-rightmove.js
- Dedup reads all landlord records and produces unified profiles
- build-lead-list.js reads deduplicated data and writes CSVs to data/exports/
