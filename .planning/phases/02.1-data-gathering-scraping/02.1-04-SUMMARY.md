## Decisions Made
- **EPC-landlord linking deferred:** EPC properties (9,639) have energy ratings but no landlord_id. CCOD/HMO properties have landlord_ids but no EPC ratings. Address-based matching needed to link them -- deferred to Phase 3 enrichment as it requires significant data engineering.
- **Review-matches needs filtering:** 289 review groups produce 231K pair combinations (O(n^2)). Phase 3 should add confidence threshold filtering or group-level review instead of pair-level.

## Deviations from Plan

None - plan executed exactly as written. The high-priority-leads CSV being empty is an expected consequence of the data structure (EPC properties are not linked to landlord records), not a deviation.

## Issues Encountered
- **0 exact match groups:** No company_number duplicates across sources (CH has 199 unique company numbers, none appear in CCOD/HMO). No UPRN overlaps (only EPC has UPRNs, CCOD/HMO/OpenRent do not). Fuzzy matching handled all dedup.
- **High-priority-leads empty:** EPC data and landlord data come from different sources with no direct linking. Properties with poor EPC ratings (9,639 total: D=7858, E=1617, F=118, G=46) all have NULL landlord_id. Requires address matching in Phase 3.
- **Review-matches very large:** 231,190 rows (18 MB) -- too large for practical manual review. Many false positives from fuzzy matching (e.g., "MR JONATHAN CALLAN" vs "MR JONATHAN WYLD"). Phase 3 should implement better filtering.
