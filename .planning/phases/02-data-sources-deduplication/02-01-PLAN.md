    3. Runs migrate-phase2.js logic (or requires it) to ensure schema is up to date
    4. Creates a readable stream with csv-parse in streaming mode: `fs.createReadStream(filePath).pipe(parse({ columns: true, skip_empty_lines: true, trim: true }))`
    5. For each row, extracts:
       - `row['Title Number']` -> titleNumber
       - `row['Tenure']` -> tenure

       - `row['Proprietor Name (1)']` -> proprietorName
       - `row['Company Registration No (1)']` -> companyRegNo
       - `row['Proprietorship Category (1)']` -> category (used to infer entity_type: "Corporate Body" -> 'ltd', "Limited Liability Partnership" -> 'llp', else 'individual')
       - `row['Proprietor (1) Address (1)']` -> proprietorAddress
       - `row['Date Proprietor Added']` -> dateAdded
    6. Filters: uses extractPostcodeOutcode(postcode) and checks POSTCODES.includes(outcode). Skips if not in target postcodes.
    7. Upserts landlord: If companyRegNo is non-empty, INSERT OR IGNORE into landlords with company_number, then UPDATE ccod_title_numbers to append title number. If no company reg, INSERT with name + source='ccod' (use normalised name for matching).
    8. Inserts property: INSERT OR IGNORE into properties with address, postcode, title_number, source='ccod', landlord_id linked.
    9. Processes in batches using a transaction every 1000 rows for performance.
    10. Logs progress every 100,000 rows: "{N} rows processed, {M} imported, {S} skipped"
    11. Final summary: "CCOD import complete: {total} processed, {imported} imported, {skipped} skipped (not in target postcodes)"
