    - Test: Two landlords with different company_numbers are NOT grouped
    - Test: Landlords with NULL company_number are not matched by company number pass
    - Test: Two properties with same UPRN (non-null) cause their landlords to be grouped

    Fuzzy matching:
    - Test: "SMITH PROPERTY LTD" and "SMITH PROPERTY LIMITED" score < 0.2 (HIGH confidence auto-merge)
    - Test: "SMITH PROPERTY LTD" and "JONES LETTINGS LTD" score > 0.4 (no match)
    - Test: "J SMITH" and "JOHN SMITH" score between 0.2-0.4 (MEDIUM confidence, flagged for review)

    Merging:
    - Test: When merging, Companies House record becomes primary (is_primary_record=1)
    - Test: When merging, EPC data wins over openrent/rightmove data for energy fields
    - Test: All merged records share the same match_group_id
    - Test: match_confidence is stored on each record in the group
  </behavior>
  <action>
    Create `scripts/lib/dedup.js` that exports three functions:

    **`findExactMatches(db)`** — Pass 1: Exact matching
    1. Query all landlords: `SELECT id, name, company_number, source FROM landlords`
    2. Query all properties with UPRN: `SELECT landlord_id, uprn FROM properties WHERE uprn IS NOT NULL`
    3. Group landlords by company_number (where non-null): all landlords sharing a company_number are a match group
    4. Group landlords by shared UPRN on their properties: if two different landlord_ids have properties with the same UPRN, those landlords are a match group
    5. Return array of match groups: `[{ ids: [1, 5, 12], matchType: 'company_number', confidence: 1.0 }, ...]`

    **`findFuzzyMatches(db, options)`** — Pass 2: Fuzzy name matching
    Options: `{ threshold: 0.3, autoMergeBelow: 0.2, rejectAbove: 0.4 }`
    1. Query landlords NOT already in an exact match group: `SELECT id, name, source FROM landlords WHERE match_group_id IS NULL`
    2. For each landlord, get their properties' postcodes to determine outcode area
    3. Group unmatched landlords by outcode area (only compare within same area)
    4. Within each outcode group, use Fuse.js:
       ```javascript
       const Fuse = require('fuse.js');
       const fuse = new Fuse(landlordsInArea, {
         keys: ['normalisedName'],
         threshold: options.threshold,
         includeScore: true,
         minMatchCharLength: 3,
       });
       ```
    5. For each landlord, search against others in same outcode
    6. Score < autoMergeBelow (0.2): HIGH confidence, add to match group
    7. Score 0.2-0.4: MEDIUM confidence, add to match group but flag for review
    8. Score > rejectAbove (0.4): no match
    9. Return: `{ autoMerge: [...groups], review: [...groups] }`

    **`mergeRecords(db, matchGroups)`** — Apply merge results
    1. Assign sequential match_group_id starting from max existing + 1
    2. For each group:
       - Determine primary record by source priority: companies-house > epc > ccod > hmo-register > openrent > rightmove
       - UPDATE all records in group: SET match_group_id = groupId, match_confidence = group.confidence
       - UPDATE primary record: SET is_primary_record = 1
    3. Return stats: `{ groupsCreated: N, recordsMerged: M, reviewFlagged: R }`

    Create `scripts/tests/test-dedup.js` that:
    - Uses temp SQLite database in os.tmpdir()
    - Seeds with test landlords across multiple sources (companies-house, epc, ccod, openrent)
    - Tests exact match on company_number
    - Tests exact match on shared UPRN
    - Tests fuzzy match on normalised names within same postcode area
    - Tests that fuzzy match does NOT match across different postcode areas
    - Tests source priority for primary record selection
    - Tests match_group_id and match_confidence are set correctly
    - Tests auto-merge vs review threshold boundary
  </action>

  <action>
    Create `scripts/build-lead-list.js` that orchestrates the full dedup + export pipeline:

    1. Requires dotenv, db.js, config.js (EXPORTS_DIR), dedup.js, normalise.js, csv-stringify/sync, fs, path
    2. Ensures EXPORTS_DIR exists: `fs.mkdirSync(EXPORTS_DIR, { recursive: true })`
    3. Runs deduplication:
       ```javascript
       const { findExactMatches, findFuzzyMatches, mergeRecords } = require('./lib/dedup');
       const exactGroups = findExactMatches(db);
       mergeRecords(db, exactGroups);
       const { autoMerge, review } = findFuzzyMatches(db, { threshold: 0.3, autoMergeBelow: 0.2, rejectAbove: 0.4 });
       mergeRecords(db, autoMerge);
       ```
    4. Generates date stamp: `const dateStamp = new Date().toISOString().split('T')[0];` (YYYY-MM-DD)
    5. Produces 5 CSV files in EXPORTS_DIR:

       **all-leads-{date}.csv** — Every landlord (primary records only where groups exist)
       Query: `SELECT l.*, GROUP_CONCAT(DISTINCT p.postcode) as postcodes, GROUP_CONCAT(DISTINCT p.current_energy_rating) as epc_ratings, COUNT(DISTINCT p.id) as property_count FROM landlords l LEFT JOIN properties p ON p.landlord_id = l.id WHERE (l.match_group_id IS NULL OR l.is_primary_record = 1) GROUP BY l.id`
       Columns: Name, Entity Type, Company Number, Sources (comma-sep), Property Count, Postcodes, EPC Ratings, HMO Licence, Registered Address

       **high-priority-leads-{date}.csv** — Has identifiable name/company AND EPC D-G
       Same query but WHERE clause adds: `AND (l.name IS NOT NULL AND l.name != '') AND EXISTS (SELECT 1 FROM properties p2 WHERE p2.landlord_id = l.id AND p2.current_energy_rating IN ('D','E','F','G'))`
       Columns: Same as all-leads plus Worst EPC Rating

       **hmo-landlords-{date}.csv** — Landlords with HMO licence numbers
       Query: Landlords WHERE hmo_licence_number IS NOT NULL, joined with their properties
       Columns: Name, Entity Type, Company Number, HMO Licence, Property Address, Postcode, Sources

       **snov-io-import-{date}.csv** — Ready for Snov.io email finding
       Query: Primary landlord records that are Ltd/LLP (entity_type IN ('ltd', 'llp'))
       Columns (per Snov.io format): First Name, Last Name, Company Name, Location, Country, Position
       - For companies: First Name = first word of company name, Last Name = rest, Company Name = full name
       - Location = "Greater Manchester, UK", Country = "United Kingdom", Position = "Director"

       **review-matches-{date}.csv** — Uncertain fuzzy matches for manual review
       Query: Landlords from `review` groups from findFuzzyMatches
       Columns: Landlord A ID, Landlord A Name, Landlord A Source, Landlord B ID, Landlord B Name, Landlord B Source, Match Confidence, Shared Postcodes
       Write directly from the `review` array data.

    6. For each CSV, uses csv-stringify/sync:
       ```javascript
       const { stringify } = require('csv-stringify/sync');
       const csv = stringify(rows, { header: true, columns: columnDefs });
       fs.writeFileSync(path.join(EXPORTS_DIR, filename), csv);
       ```

    7. Logs summary:
       ```
       Build lead list complete:
         Dedup: {exactGroups} exact groups, {autoMerge} auto-merged, {review} flagged for review
         Exports:
           all-leads-{date}.csv: {N} leads
           high-priority-leads-{date}.csv: {N} leads
           hmo-landlords-{date}.csv: {N} leads
           snov-io-import-{date}.csv: {N} leads
           review-matches-{date}.csv: {N} matches
       ```
