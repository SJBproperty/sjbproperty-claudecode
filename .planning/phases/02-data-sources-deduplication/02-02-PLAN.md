    Create `scripts/parse-hmo.js` that:

    1. Requires dotenv, db.js, config.js (POSTCODES, HMO_DIR), normalise.js, csv-parse, xlsx, fs, path
    2. Accepts command-line args: `node parse-hmo.js <file> [council-name]`
       - file: path to CSV or XLSX file (relative or absolute)
       - council-name: optional, defaults to filename stem (e.g., "stockport" from "stockport-hmo.csv")
    3. Detects file type by extension (.csv, .xlsx, .xls)
    4. Defines column mappings as an object:
       ```javascript
       const COLUMN_MAPS = {
         stockport: {
           name: ['Licence Holder', 'Landlord Name', 'Licence Holder Name'],
           address: ['Address', 'Property Address', 'Address of HMO'],
           postcode: ['Postcode', 'Post Code', 'Property Postcode'],
           licence: ['Licence Number', 'HMO Licence', 'Licence No', 'License Number'],
         },
         manchester: {
           name: ['Licence Holder', 'Landlord Name', 'Applicant Name'],
           address: ['Address', 'Property Address', 'HMO Address'],
           postcode: ['Postcode', 'Post Code', 'Property Postcode'],
           licence: ['Licence Number', 'HMO Licence', 'Licence Ref'],
         },
         _default: {
           name: ['Licence Holder', 'Landlord Name', 'Licence Holder Name', 'Applicant Name', 'Name'],
           address: ['Address', 'Property Address', 'HMO Address', 'Address of HMO'],
           postcode: ['Postcode', 'Post Code', 'Property Postcode'],
           licence: ['Licence Number', 'HMO Licence', 'Licence No', 'License Number', 'Licence Ref'],
         },
       };
       ```
    5. Reads the file header row, then attempts to match columns using the council-specific mapping first, falling back to `_default`. For each field (name, address, postcode, licence), finds the first column in the mapping array that exists in the actual headers. Logs the resolved mapping.
    6. If any of `name` or `address` cannot be mapped, exits with error: "Cannot map required columns. Found headers: [list]. Expected one of: [list]"
    7. For CSV: uses csv-parse with `columns: true, skip_empty_lines: true, trim: true`
    8. For XLSX: uses `xlsx.readFile(filePath)`, gets first sheet, converts to JSON with `xlsx.utils.sheet_to_json(sheet)`
    9. For each row:
       - Extracts name, address, postcode, licence using resolved column mapping
       - Filters by postcode: extractPostcodeOutcode(postcode), skip if not in POSTCODES
       - Upserts landlord: INSERT OR IGNORE into landlords with name=normaliseName(name), entity_type='unknown', source='hmo-register', source_ref=councilName, hmo_licence_number=licence
       - Inserts property: INSERT OR IGNORE into properties with address, postcode, hmo_licence_number=licence, source='hmo-register', source_ref=councilName, landlord_id linked
    10. Uses transaction batches of 500 rows
    11. Logs: "HMO import ({council}): {imported} imported, {skipped} skipped, {total} total"

  <action>
    **IMPORTANT:** These scripts process Apify Actor output saved as JSON files. They do NOT invoke Apify Actors — Claude MCP orchestration does that separately. These scripts are the DB insertion layer.

    Create `scripts/process-openrent.js` that:

    1. Requires dotenv, db.js, config.js (POSTCODES), normalise.js, fs, path
    2. Accepts JSON file path: `process.argv[2]` — exits with error if not provided: "Usage: node process-openrent.js <results.json>"
    3. Reads the JSON file: `JSON.parse(fs.readFileSync(filePath, 'utf8'))`
    4. For each result object (field names may vary by Actor — script should check for common alternatives):
       - Extract address from: `result.address || result.propertyAddress || result.title`
       - Extract postcode from address (last word pattern) or: `result.postcode`
       - Extract landlord name from: `result.landlordName || result.agentName || result.lettingAgent`
       - Determine self-managing: if landlordName present and no agentName, or if result contains "Private Landlord" indicator
       - Extract rent from: `result.rent || result.price || result.monthlyRent`
       - Extract listing URL from: `result.url || result.listingUrl`
       - Extract available date from: `result.availableDate || result.availableFrom`
       - Calculate void_days: if availableDate is in the past, days between availableDate and now. If future or null, set to 0.
    5. Filters by postcode (extractPostcodeOutcode, check POSTCODES)
    6. Upserts landlord (if landlord name exists): INSERT OR IGNORE into landlords with name=normaliseName(name), entity_type='individual', source='openrent'
    7. Inserts property: INSERT OR IGNORE into properties with address, postcode, listing_url, void_days, source='openrent', landlord_id if landlord exists
    8. Transaction batches of 100
    9. Logs: "OpenRent import: {imported} properties, {landlords} landlords, {skipped} skipped"

    Create `scripts/process-rightmove.js` that:

    1. Same structure as OpenRent processor but for Rightmove/Zoopla output
    2. Accepts JSON file path: `process.argv[2]`
    3. For each result:
       - Extract address from: `result.address || result.displayAddress || result.propertyAddress`
       - Extract postcode from address or: `result.postcode`
       - Extract listing URL from: `result.url || result.propertyUrl`
       - Extract listing date from: `result.listingDate || result.firstPublished || result.publishedDate`
       - Calculate void_days: days between listing date and today (longer listing = longer void)
       - Calculate listing_quality_score (0-100):
         - Photo count: `result.images?.length || result.photos?.length || 0` — score 0-40 (0 photos=0, 1-3=10, 4-7=20, 8-15=30, 16+=40)
         - Description length: `(result.description || '').length` — score 0-30 (0-100chars=0, 101-300=10, 301-700=20, 701+=30)
         - Has floorplan: `result.floorplan || result.floorplanUrl` — score 0-15 (yes=15, no=0)
         - Has EPC: `result.epc || result.epcRating` — score 0-15 (yes=15, no=0)
    4. No landlord record created (Rightmove listings typically show agents, not landlords)
    5. Inserts property: INSERT OR IGNORE into properties with address, postcode, listing_url, void_days, listing_quality_score, source='rightmove'
    6. Logs: "Rightmove import: {imported} properties, {skipped} skipped"

    Create `scripts/tests/test-openrent.js` and `scripts/tests/test-rightmove.js`:
    - Each creates temp JSON files with mock Apify output (3-5 results per file)
    - Uses temp SQLite database
    - Tests postcode filtering, field extraction, void_days calculation, listing_quality_score calculation
    - Tests graceful handling of missing fields (null landlord name, no photos, etc.)
  </action>
