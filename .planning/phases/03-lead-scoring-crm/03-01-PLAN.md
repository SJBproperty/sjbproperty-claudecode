<objective>
Link EPC properties to landlords via address matching, score all 9,545 landlords 0-100 for "tiredness" using weighted signals, classify each as BTL-suitable and/or R2R-suitable, and persist scores to the database.

Purpose: Transforms raw landlord data into a prioritised pipeline. The EPC-landlord linking step is a prerequisite -- without it, 10,677 EPC properties with NULL landlord_id cannot contribute EPC signals to scoring, and the 50+ threshold would yield far fewer than 200-300 leads. Only scored leads (50+) proceed to enrichment and CRM.

Output: EPC properties linked to landlords. Scored landlords in DB with tired_score, btl_suitable, r2r_suitable, data_signals_count columns populated. Test suites validating linking and scoring logic.
</objective>

    2. Queries all properties with NULL landlord_id grouped by postcode:

    3. For each postcode batch, queries landlords that own properties in the same postcode:
       `SELECT DISTINCT l.id, l.name, l.registered_address FROM landlords l JOIN properties p ON p.landlord_id = l.id WHERE p.postcode = ?`

    4. Uses Fuse.js (already in project via dedup.js) to fuzzy-match the unlinked property address against landlord registered addresses and existing linked property addresses in that postcode.

    5. If a match is found with score <= 0.4, UPDATE the property's landlord_id to the matched landlord.
    6. Log summary: total unlinked properties, matched count, unmatched count, match rate by postcode.

    **scripts/lib/scoring.js:** Create pure scoring functions with NO database dependency (fully testable). Export three functions:

    `scoreLandlord(landlord)` -- accepts object with optional fields: max_void_days, worst_epc, avg_listing_quality, is_self_managing, property_count. Returns `{ score: number, signals_count: number }`.

    Scoring weights (stepped, not linear):
    - **Void periods (35 points max):** null=skip. >=90 days=35, >=60=25, >=30=15, <30=5
    - **EPC rating (28 points max):** null=skip. G=28, F=24, E=18, D=10
    - **Listing quality (18 points max, INVERTED):** null=skip. <25=18, <50=12, <75=6, >=75=0
    - **Self-managing (12 points max):** false/null=skip. true=12
    - **Portfolio size (7 points max):** 0=skip. 2-5=7, 1=2, 6-8=4, 9+=0

    Create `scripts/score-landlords.js` that:

    1. Requires `./lib/db`, `./lib/scoring`, runs `migrate-phase3.js` inline (or calls it) to ensure columns exist.

    2. Queries each primary/ungrouped landlord with aggregated property signals using this SQL:
    ```sql
    SELECT l.id, l.name, l.entity_type, l.source, l.hmo_licence_number,
      MAX(p.void_days) as max_void_days,
      MIN(CASE
        WHEN p.current_energy_rating = 'G' THEN 7
        WHEN p.current_energy_rating = 'F' THEN 6
        WHEN p.current_energy_rating = 'E' THEN 5
        WHEN p.current_energy_rating = 'D' THEN 4
        ELSE NULL
      END) as worst_epc_num,
      AVG(p.listing_quality_score) as avg_listing_quality,
      MAX(CASE WHEN l.source_ref LIKE '%self-managing%' THEN 1 ELSE 0 END) as is_self_managing,
      COUNT(DISTINCT p.id) as property_count,
      -- Check if landlord ONLY has HMO properties (no non-HMO)
      CASE WHEN COUNT(DISTINCT p.id) > 0
        AND COUNT(DISTINCT p.id) = COUNT(DISTINCT CASE WHEN p.hmo_licence_number IS NOT NULL THEN p.id END)
        THEN 1 ELSE 0 END as has_only_hmo
    FROM landlords l
    LEFT JOIN properties p ON p.landlord_id = l.id
    WHERE (l.match_group_id IS NULL OR l.is_primary_record = 1)
    GROUP BY l.id
    ```

    3. Map worst_epc_num back to letter: {4:'D', 5:'E', 6:'F', 7:'G'}

    4. For each landlord, call `scoreLandlord()` and `classifyBTL()` / `classifyR2R()`.
