    **This task implements Phase A (CH officer lookup) and Phase C (mailing fallback):**

    **Phase A: Companies House Officer Lookup (no credits, free)**
    1. Query all landlords with `tired_score >= minScore` (default 50) AND `entity_type IN ('ltd', 'llp')` AND `company_number IS NOT NULL` AND `director_names IS NULL` (not yet enriched)
    2. For each, GET `{CH_API_BASE}/company/{company_number}/officers` using the same auth pattern as companies-house.js (Basic auth with API key as username, empty password)
    3. Reuse `createRateLimiter(600, 300000)` from companies-house.js (extract to shared lib or copy the pattern)
    4. Filter to active directors only: items where `resigned_on` is null/undefined
    5. Extract director names (comma-separated string), registered office address from company profile
    6. Detect service addresses using this heuristic -- if address.toLowerCase() contains any of: 'accountant', 'accounting', 'formations', 'services ltd', 'registered office', 'virtual office', 'mail box', 'mailbox', 'business centre', 'business center', 'serviced office', 'company secretar', 'nominees', 'agents' -- flag as service address
    7. For mailing_address: use registered office ONLY if NOT a service address. If service address, leave mailing_address NULL (will use property address as fallback)
    8. UPDATE landlords SET director_names, mailing_address, enrichment_source='companies-house', enrichment_date=ISO_DATE WHERE id=?

    **Phase C: Mailing Address Fallback**
    1. Query landlords with `tired_score >= minScore` AND `mailing_address IS NULL` (no registered office or was service address)
    2. For each, find their most recent property address: `SELECT address, postcode FROM properties WHERE landlord_id = ? ORDER BY created_at DESC LIMIT 1`
    3. UPDATE landlords SET mailing_address = property_address WHERE id=?
    4. These are the "no-contact leads" that are PRIME direct mail candidates per CONTEXT.md

    Log summary: CH lookups completed, service addresses flagged, mailing addresses set (registered office vs property fallback).
