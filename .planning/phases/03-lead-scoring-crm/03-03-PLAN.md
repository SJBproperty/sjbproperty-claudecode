  <action>
    **Update .env.example:** Add `NOTION_API_KEY=` and `NOTION_DATABASE_ID=` lines.

    **Create scripts/export-notion.js** with exported function `exportToNotion(db, options)` where options = `{ minScore, dryRun, batchSize }`.

    1. Query landlords with tired_score >= 50:

    ```sql
    SELECT l.*,
      COUNT(DISTINCT p.id) as property_count,
      GROUP_CONCAT(DISTINCT p.current_energy_rating) as epc_ratings,
      MAX(p.void_days) as max_void_days
    FROM landlords l
    LEFT JOIN properties p ON p.landlord_id = l.id
    WHERE l.tired_score >= 50
      AND (l.match_group_id IS NULL OR l.is_primary_record = 1)
    GROUP BY l.id
    ORDER BY l.tired_score DESC
    ```

    2. For each landlord, build a Notion page properties object:

    ```javascript
    {
      parent: { database_id: process.env.NOTION_DATABASE_ID },
      properties: {
        'Name': { title: [{ text: { content: landlord.name } }] },
        'Score': { number: landlord.tired_score },
        'Pipeline Stage': { select: { name: 'New Lead' } },
        'Lead Type': { select: { name: leadType } }, // BTL, R2R, BTL + R2R, Unclassified
        'BTL Suitable': { checkbox: Boolean(landlord.btl_suitable) },
        'R2R Suitable': { checkbox: Boolean(landlord.r2r_suitable) },
        'Entity Type': { select: { name: landlord.entity_type || 'unknown' } },
        'Property Count': { number: propertyCount },
        'EPC Rating': { select: { name: worstEPC } },
        'Void Days': { number: maxVoidDays || 0 },
        'Email': { email: landlord.email || null },
        'Phone': { phone_number: landlord.phone || null },
        'Mailing Address': { rich_text: [{ text: { content: landlord.mailing_address || '' } }] },
        'Data Sources': { rich_text: [{ text: { content: landlord.enrichment_source || landlord.source } }] },
        'Director Names': { rich_text: [{ text: { content: landlord.director_names || '' } }] },
        'Follow Up Date': { date: { start: followUpDate } }, // 3 days from now: YYYY-MM-DD
        'Company Number': { rich_text: [{ text: { content: landlord.company_number || '' } }] },
        'LinkedIn': { url: landlord.linkedin_url || null }
      }
    }
    ```

    3. **Lead type derivation:**
       - btl_suitable=1 AND r2r_suitable=0 → "BTL"
       - btl_suitable=0 AND r2r_suitable=1 → "R2R"
       - btl_suitable=1 AND r2r_suitable=1 → "BTL + R2R"
       - both 0 → "Unclassified"

    4. **follow_up_date:** Calculate as `new Date(Date.now() + 3*86400000).toISOString().split('T')[0]`

    5. **Worst EPC:** From aggregated epc_ratings, pick the letter closest to G (G > F > E > D > C > B > A)

    6. **Name splitting for director display:** If entity_type is 'ltd' or 'llp' and director_names exists, use director_names as a rich_text property. The Name (title) stays as the company name.

    7. **API calls:** Use native https.request to POST to `https://api.notion.com/v1/pages` with headers:
       - `Authorization: Bearer ${NOTION_API_KEY}`
       - `Notion-Version: 2022-06-28`
       - `Content-Type: application/json`

    8. **Rate limiting:** Notion API allows 3 requests/second. Use a simple delay of 350ms between requests.

    9. **Batch with progress logging:** Log every 10 records: "Pushed {N}/{total} leads to Notion..."

    10. **Dry run mode:** If `--dry-run` flag, build all page objects but don't POST. Write to `data/exports/notion-dry-run-{dateStamp}.json` instead. Useful for validating data before pushing.

    11. **Error handling:** If a single page create fails, log the error and landlord name, continue with remaining. Track success/failure counts.

    12. **Also export a CSV backup** to `data/exports/notion-backup-{dateStamp}.csv` with the same data (safety net — if Notion has issues, data isn't lost).

    Export `exportToNotion(db, options)` function. Run directly when `require.main === module` with CLI args: `--dry-run`, `--min-score=N`, `--batch-size=N`.

    **Create scripts/tests/test-notion-export.js:** Using temp DB pattern:
    - Insert test landlords with various scored/enriched states
    - Call the data transformation function (not the API call) with temp DB
    - Verify: correct Notion property types, follow_up_date is 3 days from today, lead_type derivation (BTL-only, R2R-only, both, neither), worst EPC calculation, director name handling for Ltd vs individual
    - Test dry-run mode writes JSON file
    - Test that only score 50+ landlords are included
